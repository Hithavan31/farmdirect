1. INTRODUCTION
1.1 Overview
FarmDirect is a comprehensive web-based platform designed to bridge the gap between farmers and retailers by eliminating middlemen. The system provides a direct marketplace where farmers can list their produce with real-time inventory management, and retailers can browse, order, and track their purchases efficiently. Built using the MERN stack (MongoDB, Express.js, React, Node.js), the platform ensures secure transactions, role-based access control, and automated inventory updates.
The platform features separate dashboards for farmers and retailers, with functionality including user authentication, product management with photo uploads, order placement with UPI/COD payment options, real-time inventory tracking, and order status management. The system is designed to be scalable, secure, and user-friendly, ensuring that both farmers and retailers can benefit from transparent pricing and direct communication.
1.2 Motivation
The agricultural supply chain in India suffers from multiple inefficiencies, primarily due to the presence of numerous intermediaries between farmers and retailers. This leads to:
  • Reduced Farmer Income: Farmers receive only a fraction of the final retail price due to multiple middlemen taking their share.
  • Higher Retail Prices: Retailers pay inflated prices, which are then passed on to consumers.
  • Lack of Transparency: Both parties have limited visibility into pricing, quality, and availability.
  • Inventory Wastage: Poor communication leads to oversupply or undersupply situations.
  • Payment Delays: Complex payment chains result in delayed payments to farmers.
FarmDirect addresses these challenges by providing a digital platform that enables direct connection between farmers and retailers. This ensures fair pricing for farmers, competitive rates for retailers, real-time inventory management, secure payment options (UPI/Cash on Delivery), and transparent order tracking for both parties.

4. SCHEMA OF PROJECT
4.1 User Schema
{ _id: ObjectId, name: String (required), email: String (required, unique, lowercase), password: String (required, hashed with bcrypt), role: String (enum: ["farmer", "retailer", "admin"]), createdAt: Date (default: Date.now), isActive: Boolean (default: true) }
4.2 Farmer Schema
{ _id: ObjectId, userId: ObjectId (ref: "User", required, unique), name: String (required), contact: String, location: String, produce: [ { name: String (required), qty: String (required), price: String (required), photo: String (Base64 encoded image) } ], createdAt: Date (default: Date.now), updatedAt: Date (default: Date.now) }
4.3 Order Schema
{ _id: ObjectId, retailerId: ObjectId (ref: "User", required), retailerName: String (required), retailerContact: String (required), farmerId: ObjectId (ref: "Farmer", required), farmerName: String (required), farmerContact: String, items: [ { produceName: String (required), quantity: Number (required), price: String (required) } ], totalAmount: Number (required), deliveryAddress: String (required), contactNumber: String (required), courierService: String (enum: ["standard", "express", "farmer-choice"]), paymentMethod: String (enum: ["upi", "cod"], required), upiTransactionId: String, status: String (enum: ["pending", "confirmed", "delivered", "cancelled"]), orderDate: Date (default: Date.now), confirmedAt: Date, deliveredAt: Date, cancelledAt: Date, notes: String }

6. BACK END DESIGN
6.1 API Architecture
The backend is built using Node.js with Express.js framework, following RESTful API principles. The server handles authentication, authorization, data validation, and business logic.
6.2 Authentication & Authorization
// JWT-based authentication middleware function auth(requiredRole) { return (req, res, next) => { const token = req.headers.authorization?.split(" ")[1]; const decoded = jwt.verify(token, JWT_SECRET); if (requiredRole && decoded.role !== requiredRole) { return res.status(403).json({ error: "Forbidden" }); } req.user = decoded; next(); }; }
6.3 API Endpoints
Authentication Routes:
POST /auth/register - Register new user (farmer/retailer) POST /auth/login - Login and receive JWT token
Farmer Routes:
GET /farmers - Get all farmers (public) GET /farmers/me - Get current farmer profile (auth) POST /farmers/profile - Create/Update farmer profile (auth) PUT /farmers/:id - Update farmer by ID (auth, owner only) DELETE /farmers/:id - Delete farmer by ID (auth, owner only)
Order Routes:
POST /orders - Place order (retailer only) GET /orders/me - Get retailer's own orders (auth) GET /orders/farmer - Get farmer's received orders (auth) PUT /orders/:id/status - Update order status (farmer only) GET /orders/:id - Get single order details (auth)
6.4 Inventory Management
The system implements real-time inventory tracking with the following features:
  • Quantity Validation: Prevents over-ordering by checking available stock before accepting orders
  • Automatic Updates: Inventory decreases automatically when orders are placed
  • Out of Stock Marking: Products with zero quantity are marked as "Out of Stock"
  • Inventory Restoration: Quantities are restored if orders are cancelled
6.5 Security Features
  • Password hashing using bcryptjs (10 salt rounds)
  • JWT tokens with 24-hour expiration
  • Role-based access control (RBAC)
  • Owner validation for edit/delete operations
  • Order privacy (retailers see only their own orders)
  • Input validation and sanitization
7. CONCLUSION AND FUTURE WORK
7.1 Conclusion
FarmDirect successfully addresses the critical challenges in the agricultural supply chain by providing a direct digital marketplace between farmers and retailers. The platform eliminates intermediaries, ensuring fair pricing for farmers while offering competitive rates to retailers. The implementation of real-time inventory management, secure authentication, and role-based access control demonstrates the system's robustness and scalability.
Key achievements of the project include:
  • Successful implementation of a full-stack web application using MERN stack technologies
  • Real-time inventory tracking with automatic updates and out-of-stock management
  • Secure authentication system with JWT and bcrypt password hashing
  • Order management system with multiple payment options (UPI/COD)
  • Product photo upload functionality with Base64 storage
  • Privacy-focused architecture ensuring order isolation between retailers
  • Responsive user interface optimized for various devices
The platform successfully bridges the gap between farmers and retailers, providing transparency, efficiency, and direct communication channels that were previously unavailable in traditional supply chains.
7.2 Future Work
Several enhancements can be implemented to further improve the platform:
Feature Enhancements:
  • Payment Gateway Integration: Integrate Razorpay or Stripe for online payments
  • Real-time Notifications: Email and SMS alerts for order status updates
  • Rating and Review System: Allow retailers to rate farmers and products
  • Advanced Search: Implement filtering by location, price range, and product category
  • Bulk Ordering: Enable ordering multiple products from multiple farmers
  • Order Tracking: Integration with courier services for real-time delivery tracking
  • Analytics Dashboard: Provide insights on sales, popular products, and revenue trends
  • Chat System: Direct messaging between farmers and retailers
Technical Improvements:
  • Cloud Storage: Migrate from Base64 to AWS S3 or Cloudinary for image storage
  • Caching: Implement Redis for improved performance
  • Mobile Application: Develop native apps for iOS and Android
  • Microservices: Split into separate services for scalability
  • GraphQL: Implement GraphQL for more efficient data fetching
  • PWA: Convert to Progressive Web App for offline functionality
Business Features:
  • Multi-language support for regional accessibility
  • Invoice generation and GST compliance
  • Subscription plans for farmers
  • Farmer verification system
  • Seasonal demand forecasting using machine learning
